ℹ️ How the DBF File Works

  1. Export your weekly sales data from your POS system as ITMSALES.xlsx.
  2. Upload it here — the app reads every item sold each day and calculates how much of each drink base was used.
  3. Each night when you calculate prep, the app automatically looks up the same day from the previous week to determine usage (e.g., tonight is Tuesday → app uses last Tuesday's sales).
  4. Safety stock is always added on top to make sure there's enough product for opening the next day.
  5. Upload a fresh file each week when new POS data is available.

⚙️ Drink Names & Daily Usage

🛡️ Safety Stock (quarts)

Minimum quarts to keep on hand as a buffer for opening the next day. Applied every night on top of the daily usage when calculating how much to make.

🥣 Batch Sizes (quarts per batch)

The size (in quarts) of each batch option. Leave blank if a size doesn't apply to that drink.
